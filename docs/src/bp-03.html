Licensing — one rule covers everything

Every shortlisted provider licenses data for personal, non-redistributed use with local storage; several (Finnhub, Tiingo most strictly) forbid showing data to anyone else, and even paid self-serve tiers stay personal-use. US real-time redistribution is further bound by exchange agreements (SIP/OPRA) at the paid tiers — another reason v1 stays end-of-day. The always-on Basic Auth wall is the compliance mechanism. Keep the FRED attribution line and the TradingView logo rendered; no public deploy previews of data pages (Vercel’s free preview authentication covers this).

Part 5

The AI layer

Three stages, five grounding rules, one hard gate. The LLM reads and writes; it never counts, never predicts, and never publishes unverified numbers.

5.2  The five grounding rules

  1. The pipeline computes all numbers; the LLM narrates them verbatim. Benchmark evals show top models collapsing from ~95% on simple financial lookups to near 0% on multivariate calculation — so arithmetic is simply not the model’s job here.
  2. Ground on fetched documents only: every input carries a doc ID; outside knowledge is forbidden; every claim must cite a doc ID — enforced via structured output schema (output_config.format / client.messages.parse), not via hope.
  3. Assume residual hallucination and gate it. Grounded-summarization hallucination rates are ~0.7–1.5% even for top models — low, not zero. Stage C exists to catch the residue; citations render inline in the UI so the user can always click through.
  4. Two-stage architecture caps the blast radius: the synthesis model never sees raw articles — only schema-validated extracts — bounding both context size and hallucination surface.
  5. No verbalized probabilities, ever. LLM-stated confidences are systematically overconfident; every probability on screen is a frequency computed from historical data by the pipeline (Research Report, Part 7).

Only Stage A is batched — it is where the volume is, and the Batch API is the right 50% discount for work nobody is waiting on. Stage B is a single call whose input depends on Stage A’s output; batching it would chain a second batch wait for a ~$0.07 saving — not worth it. Prompt caching is not the lever here either: the shared prefix (~1K tokens) sits below the minimum cacheable prefix for Haiku-class models (2–4K tokens depending on model), and the hundred calls differ in body anyway. Local models were evaluated and rejected: a CPU-only runner is slower and measurably worse, and 2026 TCO analyses put self-hosting far above a ~$7–11/month API bill.

What the LLM never does

No arithmetic. No probabilities. No direction. No “I think this stock will…”. No content that lacks a source ID. If Stage C finds an unsourced number, the briefing publishes with that line flagged — or not at all: data beats prose, and a “briefing unavailable tonight” banner over correct scan results is an acceptable outcome; a fabricated number read before the open is not.
