Part 1

Decisions at a glance

One deliberate departure from the familiar path, one architectural split, and a bill that rounds to a coffee. Everything else is the stack you already ship with.

1.1  The shape of the system

The app is read-mostly by construction: a Python pipeline does all the work in the cloud each evening after the close — ingest, scan, compute base rates, write the briefing — and the web app just renders the results. That single decision collapses most architectural questions: no always-on backend, no queues, no websockets on day one, no server for you to babysit. The one departure from your Vercel-native habits is forced by physics: Vercel Hobby caps every function at 300 seconds and fights pandas-sized Python bundles, while a scheduled GitHub Actions workflow in a private repo gives a full Python environment, 6-hour job limits, and free minutes — and the wide after-close window (the pipeline needs barely an hour of it before your evening read) absorbs its cron jitter with room to spare.

1.2  Assumptions this blueprint rests on

AssumptionConsequence if it changes
US equities — NYSE & Nasdaq — are the marketUS coverage drives every provider choice, and all timing anchors to the US session: pre-market 4:00–9:30am ET, regular 9:30am–4:00pm ET, after-hours 4:00–8:00pm ET; catalysts are the US earnings calendar and US macro prints (FOMC, CPI, jobs report). Any other market would be a separate ingest adapter behind the same repository interface — none is planned.
Single user (you), single household, non-commercialEvery data license and the Vercel Hobby terms depend on this. Showing the dashboard to others breaches provider ToS — the Basic Auth wall is the compliance mechanism.
You live on market time — Long Island, NY (America/New_York, DST observed); nothing runs on your hardwareThe pipeline runs in the cloud on UTC-fixed crons — deliberately DST-proof: your wall clock shifts twice a year, the triggers never do. 22:37 UTC lands 6:37pm EDT / 5:37pm EST — comfortably after the 4:00pm close in both regimes, so the schedule needs no seasonal edit, ever.
Watch/paper-first; no brokerage integration in v1A pedagogical stance (Research Report §6.4) — build a graded paper track record before real dollars — that also removes an entire class of security risk from v1. As a US resident you have full brokerage access, so live integration is a legitimate later phase, gated on the paper record, not on legal review.
Free-first data; pay only when specific friction appearsThe upgrade ladder in Part 4 is pre-researched so a paid migration is a same-week move, not a scramble.
Daily bars are sufficient for v1 (no intraday history)Minute bars multiply storage ~400× and change the Parquet layout; the design isolates that decision behind the ingest adapter.
Budget tolerance ≈ $15/mo now, ~$50–75/mo if upgrades fireCost table in Part 7; each upgrade is adopted singly, triggered by named friction.
Non-negotiables inherited from the Research Report

No directional forecasts anywhere in the system. Every base rate ships with sample size and a Wilson interval, computed in the pipeline — never by the LLM. The LLM narrates pre-computed numbers verbatim behind a deterministic verification gate. Signals are logged append-only from the first phase so the track record is real. No trending/most-bought surfaces. Paper-first.
